# Классификация данных, обоснование защиты и инструменты

## Классификация данных по состоянию, чувствительности и профилю риска

### Классификация по состоянию данных:
- **At Rest** - данные в покое (хранящиеся в базах данных, файлах)
- **In Transit** - данные при передаче (по сети, между системами)
- **In Use** - данные в процессе использования (в памяти, при обработке)

### Классификация по чувствительности:
- **Критический** - медицинские диагнозы, результаты анализов, финансовые данные
- **Высокий** - персональные данные (PII), медицинская информация (PHI)
- **Средний** - административные данные с частично персональной информацией
- **Низкий** - публичные данные, расписание без персональных данных

### Профиль риска:
- **Высокий риск** - данные, утечка которых может привести к серьёзным последствиям (штрафы, судебные иски, репутационные потери)
- **Средний риск** - данные, утечка которых может нарушить конфиденциальность
- **Низкий риск** - данные, утечка которых может нарушить рабочий процесс

## Автоматизированный контроль за данными

### Средства автоматизированного контроля:

#### 1. Автоматическое обнаружение и классификация данных
- **BigID** - автоматическое обнаружение конфиденциальных данных в базах данных и файловых системах
- **OneTrust** - автоматическая классификация данных по типам и уровням конфиденциальности
- **Apache Atlas** - автоматическое тегирование данных на основе политик
- **Microsoft Purview** - автоматическое сканирование и классификация данных

#### 2. Автоматический контроль доступа
- **Keycloak** - автоматическая проверка прав доступа на основе ролей и атрибутов
- **Apache Ranger** - автоматическое применение политик доступа к данным
- **Vault** (HashiCorp) - автоматическое управление секретами и ключами доступа

#### 3. Автоматический мониторинг и алертинг
- **ELK Stack** (Elasticsearch, Logstash, Kibana) - автоматический сбор и анализ логов доступа к данным
- **Prometheus + Grafana** - автоматический мониторинг метрик безопасности
- **SIEM системы** (Splunk, QRadar) - автоматическое обнаружение аномалий и подозрительной активности
- **PagerDuty / Opsgenie** - автоматические алерты при нарушении политик безопасности

#### 4. Автоматическое применение политик защиты
- **Apache Atlas** - автоматическое применение тегов и политик на основе классификации
- **Database Firewall** - автоматическая блокировка неавторизованных запросов
- **Data Loss Prevention (DLP)** - автоматическое обнаружение и блокировка утечек данных
- **Cloud Security Posture Management (CSPM)** - автоматическая проверка соответствия политикам безопасности

#### 5. Автоматический аудит и отчётность
- **Audit Logging** - автоматическое логирование всех операций с данными
- **Data Lineage Tools** - автоматическое отслеживание потоков данных
- **Compliance Automation** - автоматическая генерация отчётов для соответствия требованиям (152-ФЗ, PCI DSS)
